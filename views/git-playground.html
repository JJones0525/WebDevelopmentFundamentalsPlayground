Git

Description

  • You will use git on a routine basis(almost hourly).
  • Share code with other developers.
  • Save code locally and remotely.
    • Locally means your browser connects to local server.
    • Remotely means your browser connects to an online server.

Setting up a Git repo

Remote Set up

  1. Log into bitbucket.
  2. Find your daskboard.
  3. Click Repositories then Create Repository
  4. Give the repository the same name as what you have locally. (Not required. Just good to do) Click Create Repository.

Local Set up

  1. Open your command line/terminal.
  2. Use the cd command to navigate to your folder. For example cd desktop/projectfolder
  3. type in git init. This will set up a repository(repo) for you locally.
  4. From Bit bucket, copy and paste the line that says something like this: git remote add origin https://jamespauloconnor@bitbucket.org/jamespauloconnor/test.git. This basically points your local repository to the remote one on bitbucket.org.
  5. Type in git add . This will add all the files that have been changed.
  6. Type in git commit -m "Initial commit."
  7. Type in git push -u origin master
  8. It might ask for your bitbucket password. Enter it.
  9. Refresh and check your repository on bitbucket.org. You should see your commit message on the right side.

Routine Commits(You will do this a lot)

  1. Open your Command Line/Terminal
  2. Enter in the following commands:
    1. git add .
    2. git commit -m "Describe the code changes you made in the quotes."
    3. git push
  3. Check your bitbucket.org account to be sure it pushed.
